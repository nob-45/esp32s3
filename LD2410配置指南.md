# HLK-LD2410B 手机 App 配置指南

## 一、为什么要配置

出厂默认参数**不一定适合客厅/卧室场景**，常见问题：
- 灵敏度过低：站在 3 米外雷达检测不到 → 一直输出 EMPTY
- 无人延时过短：人稍微不动就报 EMPTY → 抖动频繁
- 探测距离过近：默认只 3-4 米，客厅覆盖不到

用手机蓝牙 App 一次性调好，参数**掉电不丢失**。

---

## 二、准备工作

### 1. 下载 App
- 官方地址：https://www.hlktech.com/Mobile/App/12.html
- **安卓**：直接扫码或从官网下载 apk
- **iOS**：App Store 搜索 **"HLKRadarTool"**（图标是深蓝色雷达波纹）

### 2. 确保雷达在通电状态
- LD2410B 的 5V 和 GND 已经接到 ESP32-S3 上
- ESP32-S3 通过 USB 上电
- **不需要断开 OUT 引脚**，蓝牙配置和 OUT 输出可以同时工作

### 3. 手机蓝牙打开
- 距离雷达 4 米以内（蓝牙有效范围）
- **不需要连 Wi-Fi**

---

## 三、连接步骤

### Step 1：搜索设备
1. 打开 HLKRadarTool
2. 首页会自动扫描附近雷达设备
3. 找到设备名 **`HLK-LD2410B_XXXX`**（XXXX 是 MAC 后 4 位）
4. 点击设备名

### Step 2：输入密码
- **首次连接必须输入密码**
- 默认密码：**`HiLink`**（区分大小写，H 和 L 都要大写）
- 输入后点击"连接"

### Step 3：进入主界面
连接成功后会看到：
- **左上角圆球**：目标状态指示灯
  - 🟢 绿色 = 无人
  - 🟣 紫色 = 有人（静止）
  - 🔴 红色 = 有人（运动）
- **中间图形区**：实时距离-能量曲线
- **底部按钮**：开始 / 停止 / 参数设置

---

## 四、关键参数配置（重点！）

### ⚠️ 配置前必须先点"停止"按钮，否则参数是灰色不能改

进入 **"参数设置"** 页面，重点调 3 个参数：

### 参数 1：最远探测距离
| 场景 | 运动最远距离门 | 静止最远距离门 | 说明 |
|:---|:---:|:---:|:---|
| **卧室（3-4m）** | 5（≈3.75m） | 5 | 覆盖整张床 |
| **客厅（4-6m）** | 8（=6m） | 8 | 最大覆盖 |
| **卫生间（2m）** | 3（≈2.25m） | 3 | 只测淋浴区 |

推荐先设 **运动=8, 静止=8**，最大化探测范围调试用。

### 参数 2：各距离门灵敏度
- 数值范围：**0-100**，值越低越灵敏（能量超过阈值就报有人）
- 出厂默认：40-50（偏保守）
- **推荐设置**：所有距离门都设 **30**（对独居老人场景更灵敏）
- 如果有宠物误触发，把距离门 1-2 的灵敏度调到 50-60

**批量修改小技巧**：多数 App 版本支持"应用到所有"按钮，一次改完 0-8 号距离门。

### 参数 3：无人持续时间
- 含义：人离开后，多少秒才切换为"无人"
- 默认：5 秒（偏短）
- **推荐**：**30 秒**（避免老人短时间静坐被误判"无人"）
- 卫生间场景可以设 **60 秒**（如厕、洗澡不动的时间更长）

### 配置完成
1. 点击 **"保存/设置"** 按钮
2. App 会提示"设置成功"
3. **参数已写入雷达 Flash**，断电不丢失

## 五、验证测试

### 1. 重新点"开始"按钮
- 看左上角圆球颜色变化
- 走远走近，看图形区距离柱变化

### 2. 断开 App，观察 ESP32 端
- App 断开蓝牙后，雷达 OUT 引脚**照常工作**
- 打开你的串口日志（read_serial.py 或 VSCode 串口监视器）
- 应该看到：
  ```
  I (xxxx) radar: presence -> OCCUPIED   ← 你走进检测范围
  I (xxxx) radar: presence -> EMPTY      ← 你离开 30 秒后
  ```

## 六、常见问题

### Q1：App 搜不到设备
- 检查雷达是否供电（用手摸模块，正常工作会**微热**）
- 手机蓝牙关掉重开
- 蓝牙功能可能被关闭了 → **在 2-3 秒内连续断电上电 5 次**可强制打开蓝牙

### Q2：密码 HiLink 不对
- 老版本固件（V1.07.22091516 以前）**不需要密码**，直接连接
- 如果你的模块是老版本，跳过密码输入即可

### Q3：配置后 ESP32 端还是一直 EMPTY
- 先看串口日志的 **`self-check` 输出**（我上一步加的诊断）：
  - 如果显示 **pin FLOATING** → 是接线问题，不是雷达问题
  - 如果显示 **pin driven LOW** → 雷达连接正常，是灵敏度问题，回到 App 把灵敏度全设为 **20**

### Q4：App 显示有人，但 OUT 引脚不给高电平
- OUT 引脚是 **3.3V** 输出（不是 5V），ESP32-S3 GPIO 电平匹配
- 用万用表测 OUT 到 GND 的电压：
  - 有人时应该 ≈ 3.3V
  - 无人时应该 ≈ 0V
- 如果电压对但 GPIO4 读不到 → 检查跳线是否松动

## 七、推荐最终参数（智慧养老场景）

| 参数 | 值 | 备注 |
|:---|:---:|:---|
| 运动最远距离门 | 8（6m） | 全屋覆盖 |
| 静止最远距离门 | 8（6m） | 关键，能测静坐/睡觉 |
| 各距离门灵敏度（运动） | 30 | 全部相同 |
| 各距离门灵敏度（静止） | 30 | 全部相同 |
| 无人持续时间 | 30s | 客厅/卧室 |
| （卫生间独立设） | 60s | 如厕时间 |

配好后蓝牙断开，参数永久生效。以后每次上电都用这套配置。
